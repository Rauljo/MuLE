# MuLE: Multilingual Language Equilibrium

MuLE investigates if Direct Preference Optimisation (DPO) can accomplish multilingual reasoning alignment at a fraction of the computing cost of the GRPO-based M-Thinker project. 

# Overview

Regardless of the input language, Large Reasoning Models (LRMs) such as DeepSeek-R1 automatically reason in English. In order to overcome this, MuLE uses DPO to fine-tune the M-Thinker Cold-Start SFT 1.5B model on self-generated multilingual preference pairings in English, French, and Portuguese before evaluating on Spanish as an out-of-domain language.

MuLE’s primary contribution is a preference-pair generation framework that is based on the multilingual equilibrium principlee, where all languages learn from each other.

# Pipeline
The pipeline consists of four steps:
### Step 1: Answer Generation
The M-Thinker Cold-Start SFT 1.5B model with vLLM is used to generate N=8 answers across English, French and Portuguese

### Step 2: Preference Pair Construction
Preference pairs for DPO training are generated by the dataset_filter.py script. The framework follows this logic:

* **Accuracy**: Checks whether the final response matches the ground truth.
* **Language Consistency**: Checks whether the output language is consistent with the input language
* **Format**: Ensures the <think>...</think>\boxed{} structure is present
* **Backtracking Count**: Checks the amount of self-correction signals
  
Outputs dpo_dataset.jsonl (preference pairs) and stats.json (dataset statistics to track in W&B).
### Step 3: DPO Training 
Uses QLoRA and Unsloth for memory efficiency.

```bash
python dpo_train_no_precomp.py \
  --dataset_file dpo_dataset_fixed.jsonl \
  --output_dir ./dpo_output \
  --beta 0.1 \
  --learning_rate 1e-6 \
  --grad_accum_steps 64 \
  --max_length 16384 \
  --wandb_run_name dpo-run1
```
### Step 4: Evaluation
Models are evaluated on two multilingual mathematical reasoning benchmarks:
* **MMATH**: 374 mathematical problems drawn from AIME 2024, AIME2025, CNMO and MATH-500
* **PolyMath**: 500 human translated problems that are classified into four difficulty tiers (low, medium, high and top) across 18 languages

Three metrics are reported:
* **LC**: Language Consistency
* **Acc**: Accuracy
* **LC&Acc** Combined metric
